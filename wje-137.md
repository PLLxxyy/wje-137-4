## 项目主要需求

### 项目概述

【Web前端】车辆管理三维可视化看板。基于 Three.js 的纯前端车辆与行驶路线 3D 可视化平台，支持车辆模型展示、路线回放、区域统计和维修日历。数据存储于 localStorage / IndexedDB。**本项目用于验证模型跨文件协同改动能力，禁止将多个职责合并进单一文件。**

### 核心实体（≥ 3 个数据模型）

1. **车辆（Vehicle）** — 字段：车牌号、车辆类型（轿车/SUV/货车/客车/特种车）、品牌型号、颜色（hex）、年份、总里程、当前状态（Active/Parked/Maintenance/Retired）、当前位置坐标
   - 定义位置：`types/vehicle.ts`

2. **行驶记录（TripRecord）** — 字段：所属车辆ID、出发地、目的地、出发时间、到达时间、行驶里程、平均速度、最高速度、油耗、路线点数组（经纬度+时间戳+速度）
   - 定义位置：`types/trip.ts`

3. **维修保养记录（MaintenanceRecord）** — 字段：所属车辆ID、维修类型（Routine/Repair/Insurance/TireChange/OilChange）、维修项目、费用、维修日期、下次保养日期、维修厂名称、备注
   - 定义位置：`types/maintenance.ts`

4. **区域统计（ZoneStat）** — 字段：区域名称、区域边界坐标数组、该区域车辆数、该区域违章次数、该区域平均速度
   - 定义位置：`types/zone.ts`

5. **报警事件（AlertEvent）** — 字段：所属车辆ID、报警类型（Speeding/Geofence/TiredDriving/Maintenance/Insurance）、报警时间、地点坐标、描述、处理状态（New/Acknowledged/Resolved）
   - 定义位置：`types/alert.ts`

### 核心页面（≥ 4 个）

1. **3D 车辆总览** `/overview` — 消费：Vehicle + ZoneStat
   - 3D 地图场景中展示所有车辆的实时位置（车辆模型），区域热力叠加，点击车辆弹出详情卡片
   - 使用 `<VehicleCard>`, `<ZoneOverlay>`, `<StatPanel>` 共享组件

2. **路线回放** `/trip-replay` — 消费：TripRecord + Vehicle
   - 选择车辆和行驶记录，3D场景中动画回放路线轨迹，同步显示速度/时间曲线
   - 使用 `<Timeline>`, `<VehicleCard>`

3. **维修日历** `/maintenance` — 消费：MaintenanceRecord + Vehicle
   - 日历视图展示所有车辆的维修计划和历史，车辆筛选，点击日历项查看详情
   - 使用 `<CalendarCell>`, `<StatusBadge>`

4. **报警中心** `/alerts` — 消费：AlertEvent + Vehicle
   - 报警列表（按类型和状态筛选），地图定位报警地点，处理报警
   - 使用 `<AlertBanner>`, `<StatusBadge>`

5. **车辆档案** `/vehicles` — 消费：Vehicle + MaintenanceRecord + TripRecord
   - 车辆列表（卡片式），车辆详情页（基本信息+里程趋势+维修记录+行驶统计）
   - 使用 `<VehicleCard>`, `<StatusBadge>`, `<StatPanel>`

### 横切关注点

1. **全局异常处理**
   - 触达文件层：Three.js渲染异常 + 数据加载失败 + IndexedDB异常 + ErrorBoundary + 错误类型

2. **主题切换（深色/浅色）**
   - 触达文件层：CSS变量 + themeStore + 组件适配 + Three.js场景光照/背景切换

### 共享枚举/常量（≥ 2 组）

1. **车辆状态 VehicleStatus**：Active / Parked / Maintenance / Retired
   - 前端：`types/enums.ts`

2. **维修类型 MaintenanceType**：Routine / Repair / Insurance / TireChange / OilChange

3. **报警类型 AlertType**：Speeding / Geofence / TiredDriving / Maintenance / Insurance

4. **车辆类型 VehicleType**：Sedan / SUV / Truck / Bus / Special

### 共享前端组件清单（≥ 3 个）

1. **`<VehicleCard>`** — 3D总览、路线回放、车辆档案
2. **`<StatusBadge>`** — 维修日历、报警中心、车辆档案
3. **`<AlertBanner>`** — 报警中心、3D总览（新报警提示）
4. **`<StatPanel>`** — 3D总览、车辆档案（统计面板）
5. **`<Timeline>`** — 路线回放

### 共享 hooks/utils 清单（≥ 2 个）

1. **`useVehicleScene`** — 封装Three.js车辆场景管理（加载模型、放置车辆、动画驱动）
2. **`useMockData`** — 生成逼真的车辆/路线/维修模拟数据
3. **`threeUtils/routePlayer.ts`** — 路线回放动画控制器

### 技术要求

- **前端端口**：18707
- **前端**：React 18 + TypeScript + Vite
- **3D引擎**：Three.js + @react-three/fiber + @react-three/drei
- **UI 库**：Tailwind CSS
- **图表**：ECharts（速度曲线、里程趋势）
- **状态管理**：Zustand
- **本地存储**：IndexedDB（Dexie.js）
- **路由**：React Router v6
- **构建工具**：Vite
